Also, when evaluating array length conditionally, `[[ ${#ARRAY[@]} -eq 0 ]]` is safer and more feature-rich than `[ ${#ARRAY[@]} -eq 0 ]`.] Action: [In high-frequency Bash validation loops, avoid subshells and process substitutions (`< <()`, `<<<`) when native loops and expansions are sufficient. Ensure `[[ ]]` is consistently used over `[ ]` for conditional tests for robust execution.]
## 2024-05-24 - [Avoid Find Subshell Overhead] Learning: [Using find process substitution (`< <(find ...)`) creates unnecessary subshell and fork overhead in loops.] Action: [Use Bash native globbing (`shopt -s nullglob; for file in ...`) and built-in checks instead of subshells to speed up execution time in bash scripts.]
## 2024-05-24 - [Batch Process Invocations] Learning: [Spawning a new process (e.g., `shellcheck`) for each file inside a `while read` loop has significant overhead. Piping a list of files to `xargs` batches these invocations, turning N forks into just 1 or a few.] Action: [Use `xargs` (with `tr '\n' '\0' | xargs -0` for filename safety) to batch external commands in Bash scripts instead of running them individually in a loop.]
## 2026-05-31 - [GitHub Actions PR Merge Commit Rebasing] Learning: [When updating repository files (like `.github/ci-status/ci-status.json`) within a GitHub Action workflow that triggered on a PR event, the runner operates on a detached HEAD containing the PR merge commit. Using `git rebase "origin/$TARGET_BRANCH"` will attempt to rebase all the merge commit's contents, causing unrelated file conflicts.] Action: [Use `git checkout "origin/$TARGET_BRANCH"` before updating the artifacts, explicitly applying changes to the actual branch instead of trying to push a rebased merge commit. Added a rebase-retry loop (max 3 attempts) so concurrent CI runs don't block each other on push conflicts.]
